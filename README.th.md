# Palette From Image

[English](README.md) | **ภาษาไทย**

เว็บแอปสำหรับดึงชุดสี (Color Palette) จากรูปภาพใดก็ได้ อัปโหลดหรือลากรูปวาง แล้วแอปจะดึงสีหลักของภาพออกมาพร้อมโค้ด HEX / RGB และสัดส่วนที่แต่ละสีครองในเฟรม คัดลอกหรือ export ผลลัพธ์ได้ในคลิกเดียว การประมวลผลทั้งหมดทำงานในเครื่องผ่านเบราว์เซอร์ด้วย Canvas รูปภาพไม่ถูกอัปโหลดไปยังเซิร์ฟเวอร์

> **สถานะ: Prototype** เป็น prototype ที่ทำงานได้จริงและมีความละเอียดสูง สำหรับใช้รีวิวงานออกแบบและส่งต่อให้นักพัฒนา ไม่ใช่เวอร์ชันสำหรับใช้งานจริงในโปรดักชัน การดึงสีทำงานได้สมบูรณ์ ส่วนเรื่องรอบข้าง (routing, persistence, analytics, การตรวจ accessibility, ตารางรองรับเบราว์เซอร์) อยู่นอกขอบเขตของขั้นนี้

## ภาพหน้าจอ

แอปมาพร้อม 2 ทิศทางการออกแบบ สลับสดได้จากปุ่มมุมขวาบน คือ **Editorial** (โทนอุ่น หัวข้อแบบ serif รูปอยู่ข้างพาเลตต์) และ **Studio** (โทนเย็น หัวข้อแบบ sans รูปอยู่เหนือกริดของสี)

## ไฟล์ที่ส่งมอบ

ไฟล์ Design Component สามไฟล์ บวก runtime หนึ่งไฟล์ ทั้งหมดอยู่ในโฟลเดอร์เดียวกัน

| ไฟล์ | หน้าที่ | จำเป็นต่อการรัน |
| --- | --- | --- |
| `Palette From Image.dc.html` | ตัวแอปจริงที่ทำงานได้ ดึงสีสด คัดลอก/export มี 2 ทิศทางการออกแบบ | ใช่ |
| `Palette Flow.dc.html` | ไฟล์นำเสนอ ภาพรวมของทุก state เรียงข้างกัน พร้อม walkthrough แบบคลิกได้ | ไม่บังคับ |
| `Palette Screen.dc.html` | คอมโพเนนต์ลูกที่ `Palette Flow` ใช้เรนเดอร์ mockup ของแต่ละ state | จำเป็นเฉพาะกับ `Palette Flow` |
| `support.js` | runtime ที่ทำให้ไฟล์ `.dc.html` เปิดในเบราว์เซอร์ได้ ต้องอยู่โฟลเดอร์เดียวกัน | ใช่ |

สำหรับการส่งมอบตัวโปรดักต์ให้นักพัฒนา ส่งแค่ `Palette From Image.dc.html` และ `support.js` ก็พอ ถ้าทีมอยากได้ภาพรวม state และ walkthrough ด้วย ให้แนบไฟล์ flow สองไฟล์ไปด้วย

## วิธีรัน

1. วางไฟล์ `.dc.html` และ `support.js` ไว้ในโฟลเดอร์เดียวกัน
2. เปิดไฟล์ `.dc.html` ในเบราว์เซอร์ได้เลย เพื่อให้โหลดฟอนต์และ asset ได้ครบ แนะนำให้เสิร์ฟโฟลเดอร์ด้วย static server (เช่น `npx serve`)

ฝังคอมโพเนนต์ลงในหน้าหรือคอมโพเนนต์อื่นด้วย

```html
<dc-import name="Palette From Image"></dc-import>
```

## 1. ตัวแอป — `Palette From Image.dc.html`

ไหลต่อเนื่องในหน้าเดียว เปิด อัปโหลด เห็นพาเลตต์ คัดลอกหรือ export หน้าจอต่าง ๆ ตามโจทย์ถูกจัดการเป็น state ภายในมุมมองเดียวนี้ ผู้ใช้จึงไม่หลุดบริบทของรูปที่เพิ่งอัปโหลด

### State ต่าง ๆ

| State | พฤติกรรม |
| --- | --- |
| Empty / Home | โซนวางไฟล์สำหรับลากวางหรือคลิกเลือกไฟล์ พร้อมรูปตัวอย่างในตัว |
| Loading | สปินเนอร์ระหว่างกำลังอ่านสี |
| Result + Export | รูปที่อัปโหลด แถบสัดส่วน และการ์ดของแต่ละสี (HEX, RGB, สัดส่วน) พร้อมปุ่ม export ในมุมมองเดียวกัน |
| Error | แจ้งเตือนเมื่อไฟล์ไม่ใช่รูปภาพ หรือมีขนาดเกิน 12 MB |

### ฟีเจอร์หลัก

- อัปโหลดด้วยการลากวาง เลือกไฟล์ หรือเลือกจากรูปตัวอย่างในตัว
- ดึงสีหลัก 4 ถึง 8 สี ปรับจำนวนด้วยสไลเดอร์ (ดึงใหม่ทันที)
- โค้ด HEX และ RGB ของทุกสี พร้อมสัดส่วนในเฟรม
- คัดลอก / export ได้ 4 รูปแบบ คือ รายการ HEX, รายการ RGB, CSS variables และ JSON
- 2 ทิศทางการออกแบบ สลับสดได้ (ดูหัวข้อการตั้งค่า)
- รองรับการแสดงผลทั้งเดสก์ท็อป (1440px) และมือถือ (390px)

### กรณีขอบที่จัดการแล้ว

- ไฟล์ที่ไม่ใช่รูปภาพ จะแนะนำให้ผู้ใช้ใช้ JPG, PNG หรือ WebP
- ไฟล์ขนาดเกิน 12 MB จะแนะนำให้เลือกไฟล์ที่เล็กลง
- รูปที่อ่านไม่ได้ จะแนะนำให้ลองไฟล์อื่น

### การตั้งค่า (props / ปรับแต่ง)

| Prop | ชนิด | ค่าเริ่มต้น | ความหมาย |
| --- | --- | --- | --- |
| `direction` | `'A'` หรือ `'B'` | `'A'` | ทิศทางการออกแบบ A = Editorial (โทนอุ่น หัวข้อ serif รูปข้างพาเลตต์) B = Studio (โทนเย็น หัวข้อ sans รูปเหนือกริดของสี) สลับสดได้จากปุ่มมุมขวาบนเช่นกัน |
| `defaultColors` | `int` 4 ถึง 8 | `6` | จำนวนสีเริ่มต้นในพาเลตต์ |

## 2. ไฟล์นำเสนอ — `Palette Flow.dc.html`

เป็นชิ้นงานแยกสำหรับรีวิวและส่งมอบ ไม่กระทบตัวแอป มี 2 โหมด สลับจากปุ่มด้านบน

- **Overview** วาง state ทุกอันเป็นเฟรมมีป้ายกำกับเรียงข้างกัน (Empty, Loading, Result + Export, Error และเฟรมมือถือ 390px) เลื่อนแนวนอนเพื่อดูทั้งโฟลว์ได้ในครั้งเดียว
- **Click-through** หน้าจอสดหนึ่งหน้าในกรอบเบราว์เซอร์ คลิกโซนวางไฟล์หรือรูปตัวอย่างเพื่อเปลี่ยนจาก Empty เป็น Loading และ Result กด "New image" เพื่อรีเซ็ต และ "Simulate an upload error" เพื่อไปยัง state Error

ทั้งสองโหมดขับเคลื่อนด้วยคอมโพเนนต์ลูกร่วมกัน คือ `Palette Screen.dc.html` ทำให้ mockup สอดคล้องกันจากแหล่งเดียว

## 3. การดึงสีทำงานอย่างไร (สำหรับนักพัฒนา)

แอปใช้วิธี median-cut quantization

1. ย่อรูปให้ด้านที่ยาวที่สุดไม่เกิน 160px วาดลงบน Canvas แล้วอ่านพิกเซลทั้งหมด
2. เริ่มจากกล่องเดียวที่บรรจุทุกพิกเซล แล้วแบ่งกล่องที่มีช่วงสีกว้างที่สุด (ถ่วงน้ำหนักด้วยจำนวนพิกเซล) ออกเป็นครึ่งตามแกนสีที่กระจายมากที่สุด ทำซ้ำจนจำนวนกล่องเท่ากับจำนวนสีที่ต้องการ
3. เฉลี่ยพิกเซลในแต่ละกล่องเพื่อให้ได้สีตัวแทน และคำนวณสัดส่วนของแต่ละสีจากจำนวนพิกเซลในกล่องนั้น
4. เรียงสีจากสัดส่วนมากไปน้อย

ค่าความสว่าง (luminance) ของแต่ละสีเป็นตัวกำหนดสีตัวอักษรบนการ์ด (ดำหรือขาว) เพื่อให้อ่านง่าย
